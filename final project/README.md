# IITB Insti-Assist — Academic Assistant

A RAG-powered academic assistant that answers questions about IIT Bombay's
official undergraduate academic rules — the academic calendar, grading and
CPI/SPI rules, registration and academic-standing categories, medical
entitlements, and medals/prizes — built entirely from IIT Bombay Academic
Office documents, with answers generated by the Google Gemini API grounded
in retrieved context.

Scope: **Academic Assistant** — academic calendar, registration & academic
standing, grading/CPI/SPI rules, examination rules, student medical
entitlement rules, and medals/academic prizes.

## How it works

```
data/*.md  ->  chunk (sliding window, paragraph-aware)  ->  embed (MiniLM)
           ->  FAISS index  ->  [query time] retrieve top-k
           ->  inject retrieved chunks into a prompt  ->  Gemini API (grounded answer)
           ->  answer + cited sources (with links to the original PDFs)
```

Retrieval finds the most relevant chunks, and the retrieved context is
injected into a prompt sent to the Gemini API, which is instructed to
answer only from that context. If nothing in the knowledge base is
similar enough to the question, the assistant skips the API call
entirely and says "I don't know" instead of guessing.

## Project structure

```
insti-assist/
├── data/           # source documents converted from official IITB Academic Office PDFs
├── src/
│   ├── ingest.py   # loads, chunks, embeds documents (.md and .pdf), builds the FAISS index
│   ├── rag.py      # retrieve / build_prompt / generate_answer / rag_answer pipeline
│   └── app.py      # Streamlit web UI
├── index/          # generated by ingest.py (FAISS index + chunk metadata)
└── requirements.txt
```

## Setup

```bash
cd insti-assist
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
```

Set your Gemini API key as an environment variable:

```bash
export GEMINI_API_KEY=your-key-here
```

Get a free key (no credit card required) at https://aistudio.google.com/apikey.
The free tier is rate-limited (requests per day/minute) but sufficient for
this project's scale.

By default the app calls `gemini-flash-latest`, an alias Google maintains
to always point at their current recommended Flash model. Google renames
and retires specific model versions frequently (e.g. `gemini-2.5-flash`
was retired for new API keys shortly after release), so the alias is used
instead of a pinned version to avoid the app breaking on its own. To pin a
specific model instead, set `INSTI_ASSIST_MODEL` before running.

## Run

Build the index once, or whenever data/ changes:

```bash
python src/ingest.py
```

This downloads the all-MiniLM-L6-v2 embedding model on first run,
embeds all document chunks, and writes index/faiss.index +
index/chunks.json.

Launch the web app:

```bash
streamlit run src/app.py
```

Or use it as a console chatbot:

```bash
python src/rag.py
```

## Data sources

All source documents are official, publicly available PDFs from the IIT
Bombay Academic Office (acad.iitb.ac.in), converted to markdown for
ingestion. Each file's original URL is embedded in the markdown (as a
`URL:` line) and carried through to the sources panel in the UI, so every
answer can be traced back to the original document.

- **Academic Calendar 2026-27** — registration windows, semester dates,
  exam schedules — `acad.iitb.ac.in/sites/default/files/Academic_Calendar_2026-27_FINAL.pdf`
- **UG Rules & Regulations — Curriculum & Registration** — credit
  structure, Minor/Honours, registration procedures, academic-standing
  categories (I-VI) and permissible credit loads — `acad.iitb.ac.in/sites/default/files/UG_RULE_BOOK.pdf`
- **UG Rules & Regulations — Examination & Grading** — grading scale,
  SPI/CPI calculation, transcript rules, ARP, exit-degree option, branch
  change/LASE — `acad.iitb.ac.in/sites/default/files/UG_RULE_BOOK.pdf`
- **Student Medical Rules** — hospital entitlement, reimbursement limits,
  medical advance procedure — `acad.iitb.ac.in/sites/default/files/student-medical-rules.pdf`
- **Rules for Award of Medals and Academic Prizes** — President of India
  Medal, Institute Gold/Silver Medals, academic prize eligibility —
  `acad.iitb.ac.in/sites/default/files/RulesforAwardofMedalsandAcademicprizesforUGandPG.pdf`

The two UG Rule Book files split one large source PDF into two
topic-coherent files (curriculum/registration vs. examination/grading) so
that retrieval returns more focused chunks instead of one giant document
diluting similarity scores.

`src/ingest.py` can also ingest `.pdf` files directly from `data/` (via
`pdfplumber`), so newer or additional official documents can be dropped
in as PDFs without manual conversion.

## Chunking strategy

Paragraph-aware sliding window: documents are split on blank lines into
paragraphs, then greedily packed into ~600-character chunks, with 150
characters of overlap carried into the next chunk. Chunk size was reduced
from the original 800 characters because these are dense rulebook/clause
text rather than course notes — smaller chunks keep individual rules
(e.g., a single grading category or a single medal's criteria) from being
diluted by neighboring clauses.

## Known limitations / what I'd improve with more time

- Fixed top-k retrieval regardless of query complexity.
- No multi-turn memory across questions.
- No live document upload at query time; the knowledge base is fixed at
  ingestion time.
- Rules and dates change over time (e.g., a new Academic Calendar is
  published every year); the knowledge base needs periodic re-ingestion
  against the latest PDFs from acad.iitb.ac.in.
- Similarity score is shown per source but not calibrated into a
  labeled confidence/groundedness indicator beyond the three-tier badge.
- Requires a valid Gemini API key and network access; there is no offline
  fallback mode. The free tier is rate-limited (requests per minute/day),
  so heavy concurrent use will need a paid tier or a different provider.

## Bonus goals implemented

- Source display with similarity scores and a direct link to the
  original official PDF, shown in an expandable panel under every
  answer, in both the console and web versions.
- Groundedness indicator: each answer is labeled Strongly grounded /
  Weakly grounded / Not grounded based on the top retrieval similarity
  score, shown as a colored badge in the web UI and inline in the
  console version.
- Citation highlighting: within each source chunk shown in the sources
  panel, the sentences most relevant to the question (by keyword overlap)
  are highlighted, so it's easy to spot the specific passage the answer
  is likely grounded in, alongside the LLM-generated answer text itself.
